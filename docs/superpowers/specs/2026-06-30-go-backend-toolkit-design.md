# Go Backend Toolkit Design

## 背景

`GLTools` 当前是一个空的 Go 工具库项目，仓库中只有 `README.md` 和 `LICENSE`。第一版目标是封装后端服务开发中高频、通用、可复用的工具能力，并提供完整中文注释和中文使用说明文档。

## 目标

- 建立标准 Go module 项目结构。
- 提供后端服务常用工具包：基础通用工具、HTTP 响应/请求、日志、配置、错误码、重试、限流、ID 生成。
- 所有导出的函数、类型、常量、变量都提供完整中文 GoDoc 注释。
- 每个包提供单元测试，关键 API 提供示例测试或 README 示例。
- 生成中文使用说明文档，覆盖安装、包说明、典型用法和注意事项。

## 非目标

- 第一版不绑定 Gin、Echo、Fiber 等具体 Web 框架。
- 第一版不封装 MySQL、Redis、Kafka、JWT、对象存储等业务基础设施。
- 第一版不引入大型配置、日志、依赖注入框架。
- 第一版不提供 CLI 命令行工具。

## 设计原则

- 标准库优先：公共 API 尽量使用 `context.Context`、`error`、`time.Duration`、`http.Client`、`http.Request`、`http.Response` 等标准类型。
- 少量稳定依赖：只允许引入长期稳定、Go 官方扩展生态中通用的依赖。限流使用 `golang.org/x/time/rate`，YAML 配置解析使用 `gopkg.in/yaml.v3`。
- 包边界清晰：避免创建 `utils` 大杂烩包，每个包只负责一类能力。
- 行为可预测：转换、重试、限流、ID 生成等工具必须明确错误返回和边界行为。
- 易于复用：不在基础包内读取全局业务配置，不依赖特定运行时环境。
- 中文文档完整：代码注释解释用途、参数、返回值、错误条件；使用文档解释实际项目中的推荐用法。

## 包能力

### glstrings

字符串工具包，提供：

- 判断字符串是否为空或空白。
- 去除首尾空白。
- 按 rune 安全截断字符串。
- 下划线命名与驼峰命名转换。

### gltime

时间工具包，提供：

- 常用时间格式常量。
- 时间格式化与解析。
- 秒级、毫秒级时间戳转换。
- 当天开始时间和结束时间计算。

### glconv

类型转换工具包，提供：

- 字符串转 `int`、`int64`、`float64`、`bool`。
- 数字和布尔值转字符串。
- 带默认值的转换函数。
- 转换失败时返回明确错误。

### glslice

切片工具包，提供：

- 判断元素是否存在。
- 字符串、整数切片去重。
- 字符串、整数切片过滤空值或零值。
- 泛型版本的 `Contains`、`Unique`、`Filter`。

### glmap

Map 工具包，提供：

- 提取 keys。
- 提取 values。
- 判断 key 是否存在。
- 合并 map，后者覆盖前者。

### glrand

随机工具包，提供：

- 生成随机数字验证码。
- 生成随机字母数字字符串。
- 使用 `crypto/rand` 作为安全随机源。

### glcrypto

加密与摘要工具包，提供：

- MD5 摘要。
- SHA256 摘要。
- HMAC-SHA256 签名。
- Base64 编码和解码。

### gljson

JSON 工具包，提供：

- JSON 序列化。
- JSON 反序列化。
- 判断字符串是否为合法 JSON。
- 将对象格式化为缩进 JSON 字符串。

### glfile

文件工具包，提供：

- 判断文件或目录是否存在。
- 创建目录。
- 读写文本文件。
- 获取文件扩展名。
- 安全拼接路径。

### glhttp

HTTP 工具包，提供：

- 统一 JSON 响应结构。
- 成功响应和失败响应构造。
- 轻量 HTTP 客户端，支持超时、Header、JSON 请求体。
- GET、POST JSON 请求封装。

该包不绑定具体 Web 框架。响应结构只负责数据表达，由使用方决定如何写入框架响应。

### gllog

日志工具包，提供：

- 基于 `log/slog` 的日志初始化。
- 支持 JSON 和 Text 输出格式。
- 支持日志级别设置。
- 支持上下文字段。

### glconfig

配置工具包，提供：

- 从 JSON 文件加载配置。
- 从 YAML 文件加载配置。
- 从环境变量读取字符串、整数、布尔值。
- 支持默认值。

### glerror

错误码工具包，提供：

- `CodeError` 类型。
- 错误码、错误消息、原始错误包装。
- 判断和提取 `CodeError`。
- 将错误转换为 HTTP 响应所需的 code/message。

### glretry

重试工具包，提供：

- 固定间隔重试。
- 指数退避重试。
- 支持 `context.Context` 取消。
- 返回最后一次错误。

### gllimit

限流工具包，提供：

- 基于令牌桶的限流器。
- 支持立即判断是否允许。
- 支持等待直到允许或上下文取消。

### glid

ID 生成工具包，提供：

- Snowflake 风格的分布式 ID 生成器。
- 支持自定义机器 ID。
- 返回 `int64` ID。
- 提供字符串 ID 便捷方法。

## 错误处理

- 函数能失败时必须返回 `error`，不得吞掉错误。
- 带默认值函数可在失败时返回默认值，但函数名必须体现默认值语义。
- 错误信息使用英文常量或简单英文文本，中文说明写在注释和文档中，避免错误值难以被程序处理。
- `glerror` 负责业务错误码表达，其他包只返回普通错误。

## 注释规范

- 所有导出标识符必须有中文 GoDoc 注释。
- 注释首词必须是被注释的标识符名称，符合 Go lint 习惯。
- 注释说明用途、参数含义、返回值含义和关键边界行为。
- 示例：

```go
// IsBlank 判断字符串在去除首尾空白后是否为空。
//
// 参数 s 表示待检查的字符串。
// 返回 true 表示字符串为空字符串或只包含空白字符。
func IsBlank(s string) bool
```

## 测试策略

- 每个包至少包含一个 `_test.go` 文件。
- 普通工具函数使用表驱动测试。
- 随机、时间、重试、限流、ID 生成等行为使用可控参数降低测试不稳定性。
- HTTP 客户端测试使用 `httptest.Server`。
- 文档示例使用 `Example` 测试覆盖关键 API。
- 最终验证命令：

```bash
go test ./...
go vet ./...
```

### README.md

README 提供：

- 项目简介。
- 安装方式。
- 快速开始。
- 包列表。
- 基础示例。
- 测试命令。

### docs/usage.md

完整中文使用说明提供：

- 每个包的用途。
- 常用函数列表。

- 推荐实践。
- 注意事项。

## 兼容性

- Go 版本使用 Go 1.22 或更高版本。
- 公共 API 在第一版发布后应尽量保持兼容。
- 新增能力优先通过新增函数或新增选项结构实现，避免破坏已有函数签名。

## 验收标准

- 仓库包含可运行的 `go.mod`。
- 所有计划内包均已实现。
- 所有导出 API 均有中文注释。
- `README.md` 和 `docs/usage.md` 均为中文文档并包含示例。
- `go test ./...` 通过。
- `go vet ./...` 通过。
